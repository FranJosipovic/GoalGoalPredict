# Release 2026-06-11

## Match detail: your prediction + substitutions
- Added a **"YOUR PREDICTION"** block at the top of the live/finished match screen showing
  your predicted score, scorer/card picks (with points earned once scored) and projected/final points.
- Added a **"SUBSTITUTIONS"** section showing who came on (▲) and who went off (▼), with minute,
  home/away aligned. Only populates for real (API-Football) matches; simulated matches don't generate subs.

## Fix: match detail page was never reached
- Tapping a **live or finished** match now opens the match detail screen (MatchLivePage) instead of
  the prediction screen, so goals, substitutions and your prediction actually show. Upcoming matches
  still open the prediction screen. (Previously the `/live` detail route existed but nothing navigated to it.)

## Leaderboard (Board tab)
- Removed the per-player points breakdown line (`⚡×7 ✓×2 ⚽pts`) under each name. Rows now show
  position, name and total points only.

## Group tabs: swipe navigation
- Swipe left/right on the tab content to move between tabs (Matches / Picks / Board / Members / Rules).
  Mostly-vertical gestures (scrolling) are ignored.

## Admin: manual match data sync
- New **"Match Events & Lineups"** section in Admin → Sync & Compare. "Load matches" lists **finished**
  real matches with their stored counts (events G/C/S and lineup players). Per-row buttons:
  - **Sync events** — pulls status + goals + cards + substitutions, upserts only what's missing.
  - **Sync lineups** — pulls starting XI + bench (no-op if lineups already stored).
  Both are idempotent and safe to re-run.

---

## Technical
- New `MatchSubstitution` entity + `match_substitutions` table (migration `AddMatchSubstitutions`,
  auto-applied on startup). FK to players is nullable; unknown bench players are stored as null (shown "Unknown").
- API-Football substitution events: `assist` = player IN, `player` = player OUT
  (`ApiFootballClient.GetSubstitutionEventsAsync`).
- `PollLiveMatch` now also ingests substitution events (dedup by `(match_id, api_event_order)`).
- `GetMatches.GetDetailAsync` includes substitutions and uses `AsSplitQuery()` to avoid cartesian blow-up.
- Routing: `MatchesTab`/`PicksTab` `onMatchClick(matchId, openDetail)` where `openDetail = live || finished`;
  `GroupDetailPage` routes to `/groups/:id/match/:matchId/live` when `openDetail`.
- Admin endpoints:
  - `GET /admin/matches` — finished API-Football matches with event + lineup counts.
  - `POST /admin/matches/{id}/sync-events` — runs `PollLiveMatch`.
  - `POST /admin/matches/{id}/sync-lineups` — runs `SyncLineups`.
