# Release 2026-06-12

## Predictions grouped by team (everywhere)
- A user's scorer/card picks are now grouped under each team (with team crest + code) instead of one
  flat name list. Applied everywhere predictions show: match detail ("YOUR PREDICTION"), the Picks tab
  (your card + revealed group picks), and the player history page.
- Layout is **two columns**: home team on the left, away team on the right, each with its picks listed
  vertically beneath the crest (away column right-aligned). Empty side shows a "—".
- In the Picks tab's expanded group picks, each player's team grouping now sits **below** their name +
  predicted score (was overflowing to the right).

## Match detail: cleaner, focused view
- **Other users' predictions removed** from the match detail screen — it now shows only your own
  prediction. (Group picks are still available in the Picks tab.)
- **Events unified into one list** — goals, cards and substitutions now appear together in a single
  "MATCH EVENTS" timeline sorted by minute, instead of separate "GOALS" / "SUBSTITUTIONS" sections.

## Fix: VAR-disallowed goals lingered forever
- When VAR overturns a goal (or card), API-Football **drops** that event from the match feed.
  Our sync was append-only, so the now-phantom goal stayed in the database and kept showing on the
  match detail screen even though the scoreline was correct. Event sync now **reconciles**: it adds
  new events *and removes* events that are no longer in the API feed (e.g. disallowed goals/cards).

## VAR decisions now captured + shown
- API-Football pushes goals **live as they happen** and then rewrites the feed when VAR overturns
  them — it deletes the goal event and adds a `Var` event like "Goal Disallowed - offside". We used
  to discard those `Var` events entirely. They're now stored and shown in the **MATCH EVENTS**
  timeline (❌ for a disallowed/cancelled goal, 📺 for other VAR reviews), e.g. "VAR: Goal Disallowed
  - offside".

## Fix: lineup sync crashed on players missing from our squad
- Syncing lineups for a match threw a foreign-key error when the starting XI/bench contained a
  player we hadn't synced into our `players` table yet. Lineup sync now fetches that team's squad
  on demand to fill the gap, and skips (with a warning) any player the API squad still doesn't list,
  instead of failing the whole sync.

## Admin: compare match events (DB ↔ API)
- New **"Compare events"** button per match in Admin → Sync & Compare → Match Events. Shows goals,
  cards, substitutions **and VAR decisions** stored in our DB next to what the API currently returns,
  each row flagged **in sync** / **DB only — not in API (disallowed?)** / **API only — missing in DB**.
  Makes discrepancies (like a VAR-disallowed goal) obvious before running a sync. Running "Sync events"
  with the diff open refreshes it in place. Match list event counts now read G/C/S/V.

## Technical
- `PollLiveMatch` goals/cards/VAR now reconcile against the API feed by natural key instead of
  append-only on the positional `ApiEventOrder` (which shifts when an earlier event is removed).
  Stale rows not present in the current feed are deleted. New rows get a monotonic `max(order)+1`
  so the `(match_id, api_event_order)` unique index can't collide with kept rows. Goal/card key is
  `(team, player, minute, extraMinute, type)`; VAR key is `(team, minute, extraMinute, detail)` (the
  player is incidental and may be nulled). Substitutions remain append-only.
- New `MatchVarDecision` entity + `match_var_decisions` table (migration `AddMatchVarDecisions`,
  auto-applied on startup). Nullable FK to `players` (unknown ids nulled like subs). `GetVarEventsAsync`
  fetches the **unfiltered** `fixtures/events` (API-Football's `type` filter only accepts
  goal/card/subst, not Var) and picks out `type=="Var"`. Surfaced in `MatchDetailDto.VarDecisions` (`VarDecisionEventDto`)
  and rendered as a `var` event kind in the `MatchLivePage` timeline.
- `AdminCompareService.CompareMatchEventsAsync(matchId)` → `MatchEventsCompare` (now 4 groups incl.
  `Var`, per-group DB/API rows with `inDb`/`inApi` flags); exposed at
  `GET /api/admin/compare/matches/{id}/events`.
- Frontend: `compareMatchEvents(id)` client fn; `EventsCompareView` expandable row in `AdminSync`;
  `VarDecisionEvent` type.
- `ScorerPickDto` / `CardPickDto` gained a `TeamId` (populated from `Player.TeamId` in
  `GetGroupPredictions` and `GetMyPredictions`); mirrored in the frontend `ScorerPick` / `CardPick` types.
- New shared component `components/PicksByTeam.tsx` (groups picks by team) replaces the bespoke chip
  rendering in `PicksTab` (ScorerChips/CardChips + group panel), `PlayerDetailPage`, and the
  `MatchLivePage` "YOUR PREDICTION" block.
- `MatchLivePage`: removed the GROUP PREDICTIONS section; merged goals/cards/subs into one sorted
  `events` timeline (`.events-timeline` / `.ev-*` styles).
